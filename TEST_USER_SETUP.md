# テストユーザーの追加方法

## テストユーザーの設定場所

テストユーザーは「OAuth同意画面」の設定ページから追加します。

## 手順

### 方法1: Google Cloud Consoleから直接アクセス

1. [Google Cloud Console](https://console.cloud.google.com/)にアクセス
2. プロジェクト「Gemini Project」が選択されていることを確認
3. 左サイドバーから以下を選択：
   - **「APIとサービス」** → **「OAuth同意画面」**
   
   または
   
   - 検索バーに「OAuth同意画面」と入力して検索

### 方法2: 現在のページから移動

現在「Google Auth Platform / 概要」ページにいる場合：

1. 左サイドバーの**「対象」**(Audience)をクリック
   - これでOAuth同意画面の設定ページに移動します
2. ページを下にスクロール
3. **「テストユーザー」**セクションを探す

### テストユーザーの追加

1. 「テストユーザー」セクションを開く
2. **「+ ユーザーを追加」**ボタンをクリック
3. 自分のGoogleアカウントのメールアドレスを入力
   - 例: `your-email@gmail.com`
4. **「追加」**をクリック

### 確認

追加したメールアドレスが「テストユーザー」のリストに表示されれば完了です。

## 重要事項

- **テストユーザーとして追加したメールアドレスでログインする必要があります**
- テストユーザー以外のアカウントではログインできません
- 本番環境で使用する場合は、OAuth同意画面を「公開」する必要があります

## トラブルシューティング

### 「テストユーザー」セクションが見つからない場合

1. 左サイドバーで「APIとサービス」を確認
2. 「OAuth同意画面」を直接検索
3. URLで直接アクセス:
   ```
   https://console.cloud.google.com/apis/credentials/consent
   ```

### テストユーザーを追加してもログインできない場合

- 追加したメールアドレスが正しいか確認
- Supabaseの設定が完了しているか確認
- ブラウザのキャッシュをクリアして再試行
